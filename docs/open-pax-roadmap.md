# Open-Pax → «как оригинал»: gap-анализ и роадмап

**Дата:** 2026-07-17
**Объекты:** оригинал — [paxhistoria.co](https://www.paxhistoria.co) (+ wiki.paxhistoria.co), клон — [github.com/mopga/Open-Pax](https://github.com/mopga/Open-Pax) (HEAD, ~74 коммита)
**Цель:** локальная/персональная игра уровня оригинала со своей LLM (локальная модель или облачный API-ключ, настраиваемо).

---

## 1. TL;DR

Open-Pax сейчас — рабочий MVP-прототип с правильно угаданной архитектурой промптов (переменные `PLAYER_POLITY`, `GRAND_MAP_DESCRIPTION_NO_CITY`, `ALL_EVENTS_WITH_CONSOLIDATION` повторяют оригинал), но с фундаментальными проблемами: **жёсткая привязка к одному провайдеру (MiniMax), два параллельных игровых движка, кастомный промпт мира физически не доходит до LLM, карта из прямоугольников-заглушек, нет дипломатических чатов и консолидации контекста, сломан фронтенд-билд**.

До «как оригинал» — 7 этапов. Критический путь: **стабилизация → LLM-слой с провайдерами → единый движок симуляции → дипломатические чаты → реальная карта → пресеты → UX-полировка**. Монетизация, аккаунты, UGC-каталог, арена и модерация оригинала для локальной игры **не нужны** и сознательно исключены из скоупа.

---

## 2. Что представляет собой оригинал (эталон)

Pax Historia — браузерная песочница альтернативной истории, где **вся игровая логика выполняется LLM**: игрок пишет действия на естественном языке, ИИ симулирует реакцию мира и перерисовывает карту. YC W26, альфа с августа 2025, ~35K DAU, 4000+ публичных пресетов.

### Core loop (это и есть «как оригинал»)

1. **Чтение событий** — после каждого прыжка ИИ выдаёт ленту событий (войны, границы, дипломатия, экономика).
2. **Советник (Advisor)** — ИИ-ассистент: советы и саммари состояния игры.
3. **Действия (⚡)** — свободный текст; кнопки **Brainstorm** (ИИ предлагает варианты) и **Enhance** (ИИ улучшает формулировку). Можно несколько действий за ход.
4. **Дипломатия (💬)** — чаты 1-на-1 и групповые с ИИ-нациями; содержание чатов влияет на генерацию событий; ИИ может сам инициировать чат.
5. **Прыжок вперёд (»)** — «к следующему важному событию» или фикс: 1 неделя / 1 / 3 / 6 мес / 1 год / произвольно. Во время симуляции: **Save** (сохраниться посреди) и **Intervene** (прервать на событии).
6. Повтор. Есть **Rewind** (откат хода).

### Ключевые механики-эталоны

- **Пресет = карта (регионы+страны+фичи) + дата старта + «The World Before…» (лор для ИИ) + «Simulation Rules» + кастомные промпты.** Поле «About» ИИ не читает — только для игроков.
- **Карта**: MapLibre GL + PMTiles; регионы 4 типов (land/coastal/ocean/strait); map features с тегами `battalion` (военные юниты, видимы ИИ, двигаются/захватывают), `city` (косметика, невидимы ИИ), `capital` (звезда). **ИИ работает по именам, не по координатам** — движок резолвит имена в геометрию.
- **Создание игры**: выбор страны (рекомендованные / все / custom — «играй за что угодно») → сложность (5 уровней, реализована **текстовыми блоками в промптах** — полные тексты есть в вики) → качество ИИ (тиеры Light/Pro/Max + выбор конкретной модели).
- **Промпт-система**: 12 категорий промптов (Jump Forward, Auto Jump, Chat with User, Chat with Advisor, Actions/Brainstorm, Next Speaker, Description→Action, Event Consolidator, Catalysts ×3, Game Master). JSON-вывод (со схемой) у прыжков и чатов → движок исполняет: спавн/движение батальонов, создание стран, передача регионов, старт чатов, текст событий. Helpers — интерполяция `${VAR}`. Промпты: Default (системные) vs Local (пресет/игра).
- **Консолидация контекста**: с ~15 раунда, чанками по 5 раундов — саммари истории вместо сырого хвоста; к 80-му раунду запрос ~100K токенов.
- **Разные модели для разных механик** (дешёвая для чатов, дорогая для прыжков).

## 3. Текущее состояние Open-Pax

**Стек:** монорепо (npm workspaces): React 18 + TS + Vite + Zustand + mapbox-gl (фронт, :5173); Express + better-sqlite3 (бэк, :8000; папка зовётся `backend-nest`, но NestJS нет). LLM — только MiniMax (`MiniMax-M2.5`) через нативный `https`. Запуск `npm start`. Тестов, CI, линт-конфига нет.

**Уже реализовано (сохраняем и развиваем):**
- Редактор карт (рисование полигонов, объекты, сохранение).
- Два пути создания игры: кастомная карта → CreateWorld; шаблоны (`cold_war_1951`, `modern_world`) с LLM-генерацией начального состояния стран (BalanceAgent, батчи по 6).
- Очередь действий с LLM-подсказками, ручным вводом, типами действий; пайплайн «конвертер действий (batch) → симуляция (time-rewind) → применение worldChanges».
- Time-skip (неделя…год/кастом), снапшот-сохранения в SQLite, восстановление сессий после рестарта, graceful shutdown.
- RelationshipMatrix (ally/neutral/hostile) + панель дипломатии; NPC с личностями; SSE-канал событий.
- Промпты — верблюжья копия референсных промптов оригинала (simulation/converter/advisor/suggestions/narration) с теми же переменными.

**Критические баги (вычитаны статически, требуют проверки при запуске):**

| # | Баг | Файл |
|---|-----|------|
| 1 | **Кастомный промпт мира не доходит до LLM** — `buildGameData()` отдаёт `basePrompt: ''`; `WORLD_BEFORE_ROUND_ONE_TEXT` всегда дефолт. `STARTING_ROUND_DATE` «плывёт» каждый ход | `game-session.ts` |
| 2 | Сброс даты при рестарте сервера (`reloadActiveSessions` игнорирует `game.current_date`) | `session-registry.ts` |
| 3 | `bulkUpsert` отношений сломан: `ON CONFLICT(world_id, from_region_id, to_region_id)` без UNIQUE-индекса в схеме → 500 на ходах с дипломатией | `relationship repository` + `database.ts` |
| 4 | Рассинхрон ключей: отношения по кодам стран (`USA`), движок/UI — по owner (`ai-USA`) и id регионов → матрица дипломатии пуста; личности NPC захардкожены под `ai-1..4` | инициализация / `npc-agents.ts` / `DiplomacyPanel` |
| 5 | **mapChanges от LLM почти никогда не применяются**: промпт требует `regionId`, а LLM получает только имена регионов | `simulation.ts` / `applyWorldChanges` |
| 6 | `DiplomacyPanel` слушает `window`-событие `turn_complete`, которое никто не диспатчит | фронт, SSE-хук |
| 7 | В LLM-ветке NPC захват региона пишет `owner = npcRegionId` (id региона вместо владельца); «соседи» = первые 5 регионов, не реальные | `game-session.ts` |
| 8 | Сборка фронта сломана (`tsc && vite build` падает на implicit any в сторах); невалидный цвет `#FFOA00` у Эквадора; мусорный `__init__.py` в TS-проекте | `gameStore.ts`, `uiStore.ts`, `countries.json` |

**Архитектурные пробелы:**
- Два параллельных движка (LLM-очередь vs детерминированный `applyTurn`) с разной семантикой.
- Границы регионов (`borders`) никогда не вычисляются → в legacy-движке любая атака невалидна; в очереди валидации нет вообще.
- Дипломатические чаты отсутствуют полностью (конвертер умеет `type: 'chat'`, но ни эндпоинтов, ни хранилища, ни UI).
- История событий растёт без консолидации → неизбежный раздув контекста.
- Нет сложности, rewind/Intervene, выбора модели, «void unrealistic action».
- GeoJSON стран — прямоугольники-bbox; кастомные SVG растягиваются на весь глобус.
- Нет стриминга, retry, таймаутов у LLM-клиента; без ключа в нарратив утекает строка-заглушка `[LLM not configured]`.
- In-memory LLM-кэш без лимита (TTL 5 мин): одинаковое действие в течение 5 минут вернёт тот же нарратив.
- SSE-прогресс не работает в основном флоу (очередь ничего не broadcast'ит); `sse.ts` захардкожен `localhost:8000`.
- Монолитный `App.tsx` (1372 строки), мёртвый код, сохранение/загрузка через `prompt()` и грузится только `saves[0]`.

## 4. Gap-анализ: оригинал vs Open-Pax

| Подсистема | Оригинал | Open-Pax | Гэп |
|---|---|---|---|
| Действия (свободный текст) | ✅ + Brainstorm + Enhance | ✅ очередь + подсказки | Малый: добавить Enhance (есть промпт converter) |
| Прыжок вперёд | ✅ auto/fixed, Save/Intervene, события порциями | ✅ time-skip | Средний: auto-jump, Intervene, показ событий по одному |
| Дипломатические чаты | ✅ 1-на-1 и групповые, влияют на события | ❌ отсутствуют | **Критический** |
| Советник | ✅ с контекстом игры | ⚠️ промпт есть, история пустая | Малый-средний |
| Сложность | 5 уровней через промпт-блоки | ❌ | Средний (тексты уровней есть в вики) |
| Консолидация истории | ✅ с 15 раунда, чанки по 5 | ❌ | **Критический** (иначе длинные кампании умрут по контексту/цене) |
| Map changes от ИИ | ✅ регионы/батальоны/страны по именам | ⚠️ промпт есть, применение сломано | **Критический** (баг №5) |
| Реальная геометрия карты | ✅ регионы 4 типов, фичи battalion/city/capital | ❌ bbox-прямоугольники, типы объектов другие | **Критический для вида «как оригинал»** |
| Пресеты (пакет: карта+лор+правила+промпты) | ✅ каталог, форки, версии | ⚠️ 2 захардкоженных шаблона | Большой (локально достаточно файловых пакетов) |
| LLM-провайдеры | ~30 моделей, тиры, BYOK | ❌ только MiniMax | **Критический** (главная цель юзера) |
| Выбор модели / тиры | ✅ Light/Pro/Max + список | ❌ | Средний |
| Создание игры (страна/custom/сложность) | ✅ | ⚠️ частично | Средний |
| Rewind / откат | ✅ | ❌ | Малый |
| Генерация мира LLM | (авторство пресетов ручное) | ✅ BalanceAgent | У клона даже есть фича, которой нет в оригинале |
| Мультиплеер | ✅ бета | ❌ | Вне скоупа локальной игры (опционально, поздно) |
| Аккаунты/токены/Stripe/UGC-каталог/Arena/модерация | ✅ | ❌ | **Осознанно пропускаем** |

## 5. Рамка скоупа

**Копируем (суть «как оригинал» для одиночной локальной игры):** core loop целиком; дипломатические чаты; советник; сложность; тиры/выбор модели; консолидацию; JSON-схему событий с исполнением движком; пресеты как пакеты; реальную карту; rewind/Intervene; Brainstorm/Enhance.

**Не копируем:** регистрацию/Google-auth, токены и Stripe, Patron, каталог UGC и лидерборды, Pax Arena, модерацию флагов, облачные сейвы, мобильную адаптацию, i18n интерфейса (язык контента — настройка). Мультиплеер — опциональный этап в самом конце.

**Принцип оригинала, который надо украсть целиком:** *«слова → действие → реакция», никаких заскриптованных веток*. Детерминированный движок оставляем только как валидатор/исполнитель (проверка ресурсов, применение mapChanges, вычисление соседей), а не как источник исходов.

## 6. Роадмап

Оценки — в условных размерах: S (вечер), M (2–4 вечера), L (неделя+). Этапы 0–2 — критический путь, дальше можно параллелить.

### Этап 0 — Стабилизация фундамента (S–M)

Цель: проект собирается, запускается, ключевые баги закрыты, есть дым-тесты.

1. Починить сборку фронта (`tsc` implicit any в `gameStore.ts`/`uiStore.ts`), удалить мёртвый код (`CountryPanel`, `TimeJumpModal`, legacy-агенты, `__init__.py`).
2. Баги №1–8 из таблицы (особенно №1 — без него лор мира не работает вообще, и №5 — без него карта не реагирует на ИИ).
3. Завести `frontend/.env.example` (VITE_MAPBOX_TOKEN), задокументировать `MINIMAX_BASE_URL`/`LLM_MODEL`.
4. Дым-тесты: создание игры из шаблона → 3 действия → прыжок → сейв/лоад (vitest на бэке, минимум на фронте). Без этого рефакторинг этапов 1–2 — вслепую.
5. Единая конвенция идентификаторов: **polityId = код страны** (`USA`), регион `regionId = <worldId>_<code>`, владелец региона = polityId. Пройтись по NPC/дипломатии/UI (закрывает корень багов №4, №7).

**Критерий готовности:** `npm run build` зелёный; новая игра на `cold_war_1951` с кастомным лором — лор реально влияет на события первых ходов; захват региона через действие меняет цвет на карте.

### Этап 1 — LLM-слой: провайдеры, локальные модели, надёжность (M)

Цель: главная фича юзера — «своя LLM или облачный ключ, настраиваемо».

1. Интерфейс `LLMProvider` (уже запланирован в `docs/PLAN.md` Phase 3.2 — реализовать):
   ```ts
   interface LLMProvider {
     generate(req: ChatRequest): Promise<string>;
     stream?(req: ChatRequest): AsyncIterable<string>;
   }
   ```
2. Провайдеры:
   - **OpenAI-compatible** (baseURL+key+model) — закрывает OpenAI, OpenRouter, Together, DeepSeek, GLM, Kimi, а главное — **Ollama и LM Studio** (у обоих OpenAI-совместимый `/v1`). Это один адаптер на 90% сценариев «своя модель / свой ключ».
   - **Anthropic** (нативный, для Claude Max-тира).
   - **MiniMax** — текущий код переписать под интерфейс (и заменить нативный `https` на fetch/axios с таймаутом).
3. Конфиг `llm.config.json` (или UI-настройки, позже): **модель per-механика** — `jump`, `chat`, `advisor`, `suggestions`, `converter`, `consolidation`. Это прямой аналог тиров Light/Pro/Max оригинала: дефолт-наборы «эконом/баланс/макс» + ручное переопределение. Пресеты конфига: «Всё локально (Ollama)», «OpenRouter», «Смешанный».
4. Надёжность: таймаут (60–120с), retry с backoff (2–3), нормальная ошибка в UI вместо заглушки в нарратив, лимитированный LRU-кэш (или отключить кэш для симуляции — см. баг «одинаковый нарратив»), user-сообщение не пустое (сейчас весь промпт в system, user='') — часть моделей на этом деградирует.
5. Стриминг хотя бы для прыжка и чатов (SSE уже есть на фронте).

**Критерий готовности:** одна и та же кампания играется на (а) Ollama с локальной моделью, (б) любом OpenAI-compatible облачном ключе; смена модели для чатов не трогает модель прыжков; при недоступной модели — понятная ошибка, а не мусор в игре.

> Практическая заметка: для локальных моделей 8–32B контекст 1M как в оригинале недостижим — поэтому этап консолидации (№2) для локального сценария ещё важнее, чем для оригинала. Целевой бюджет: ≤16–32K токенов на прыжок.

### Этап 2 — Единый движок симуляции (L)

Цель: один игровой движок вместо двух, воспроизводящий механику оригинала.

1. **Слить два движка.** Основной путь — LLM-симуляция (time-rewind промпт): действия → конвертер → симуляция → JSON → исполнение. Из legacy `SimulationEngine` забрать: валидацию ресурсов (`assertCanAfford`), стоимость действий, `RelationshipMatrix`. Сам `applyTurn`-путь удалить.
2. **Резолвинг по именам (принцип оригинала).** LLM получает описание карты по именам и возвращает имена; движок резолвит имя → регион/страну (fuzzy match по словарю имён, в т.ч. русских/английских алиасов). Закрывает баг №5 системно. Поддержать override-форматы как у оригинала: `random`, `coastal`, `west/east/north/south`, `target X`.
3. **JSON-схема ответа симуляции** (расширить текущую): `events[]` (с датами), `mapChanges[]` (transfer_region по имени, spawn/move battalion, create_polity), `worldChanges`, `startChat` (ИИ инициирует дипломатию), `voided` (отклонённое нереалистичное действие с пояснением). Парсинг — не regex, а structured output / JSON mode где провайдер умеет + строгий ремонт JSON (jsonrepair-подход).
4. **Консолидация истории** (механика оригинала): с раунда N=15, каждые 5 раундов — LLM-саммари в `consolidated_history`; `ALL_EVENTS_WITH_CONSOLIDATION` = саммари + последние 5–10 сырых раундов. Настройки порога — в конфиг (для локальных моделей понизить).
5. **Сложность**: 5 уровней, текстовые блоки в промпты прыжка и чатов (тексты всех уровней есть на wiki.paxhistoria.co — страница Difficulty). Выбор на экране создания игры.
6. **Auto-jump** («к следующему важному событию») и показ событий **по одному** с кнопками Next/Save/**Intervene** (прервать на событии = откатить остальные события пачки).
7. **Rewind**: снапшоты уже есть — добавить «откат на ход назад».
8. Реальное вычисление соседства регионов (turf.js intersects/touches один раз при создании мира → таблица `borders`) — нужно и для валидации, и чтобы NPC не воевали через океан.

**Критерий готовности:** кампания на 30+ ходов: контекст не раздувается (лог токенов), границы на карте двигаются, нереалистичное действие («захватить мир») корректно «войдится», сложность ощутимо меняет исходы, Intervene/Rewind работают.

### Этап 3 — Дипломатические чаты и живой Советник (M–L)

1. Хранилище чатов (таблицы `chats`, `chat_messages`), эндпоинты, UI-панель 💬 (список чатов + лента).
2. Промпт `Chat with User` (структура из вики оригинала): персонаж лидера, учёт отношений, сложности, последних событий; JSON-вывод: текст реплики + опциональные эффекты (смена отношений, согласие на сделку → mapChange при следующем прыжке).
3. Групповые чаты: промпт `Next Speaker` (ИИ выбирает очерёдность), кнопка «Speak».
4. Чаты попадают в контекст прыжка (`CHATS_NON_CONSOLIDATED_ROUNDS`) — договорённости реально влияют на события.
5. ИИ инициирует чат по итогам событий (ультиматумы, предложения мира) — флаг `startChat` из этапа 2.
6. Советник: POST с историей диалога + полное состояние игры; кнопки-саммари («оцени моё положение»).

**Критерий готовности:** можно договориться с ИИ-Германией о разделе Польши в чате — и после прыжка увидеть это в событиях и на карте.

### Этап 4 — Карта «как в оригинале» (L)

1. **Реальная геометрия:** Natural Earth (10m/50m admin-0, при желании admin-1) → GeoJSON-регионы для шаблонов вместо bbox-прямоугольников. Исторические границы (1914/1936/1951) — либо готовые пакеты, либо ручная разметка в редакторе поверх современной подложки.
2. **Типы регионов** land/coastal/ocean/strait и **map features** с тегами `battalion`/`city`/`capital` (семантика оригинала: батальоны видны ИИ и захватывают, города — косметика, столица — звезда).
3. Переезд фронта с mapbox-gl на **MapLibre GL** (убирает зависимость от токена Mapbox — для локальной игры это лишнее) или остаться на mapbox-gl, но сделать токен необязательным (подложка OSM/Carto).
4. Заливка регионов цветом владельца (data-driven стили), отключение границ/лейблов (Display Options), поиск по странам/регионам/фичам, перелёт камеры к столице.
5. Кастомные карты: нормальная привязка координат (проекция холста → geo по аффинному преобразованию с калибровочными точками), а не растяжение на весь глобус.
6. Локальные флаги (своя папка флагов по polityId, без flagcdn — офлайн).

**Критерий готовности:** шаблон WW2/Modern выглядит как настоящая политическая карта; батальоны ставятся/двигаются действием; захват столицы виден.

### Этап 5 — Пресеты как пакеты (M)

1. Формат пресета (одна папка / zip): `preset.json` (название, дата, категория, описание для игроков) + `world_before.md` (лор для ИИ) + `simulation_rules.md` + `prompts/*.md` (кастомные локальные промпты, переопределяют дефолтные) + `map.geojson` + `flags/`.
2. Менеджер пресетов: список локальных пресетов, создание (Blank Canvas / форк существующего / из текущей игры «сохранить как пресет»), импорт/экспорт zip.
3. Экран пресета как в оригинале: About / The World Before / Simulation Rules / карта регионов / Play Now / Copy.
4. Конвертер существующих шаблонов (`cold_war_1951`, `modern_world`) в новый формат.

**Критерий готовности:** пользователь может создать пресет «Средневековая Европа 1444» с лором и правилами, поделиться zip-файлом, другой пользователь импортирует и играет.

### Этап 6 — UX-полировка и опциональное (M, по вкусу)

1. Разобрать монолит `App.tsx` на роуты/экраны (react-router уже в зависимостях): Games / Presets / Game.
2. Игровой экран по образцу оригинала: карта на весь экран; дата-пилюля + » справа сверху; 💬⚡🔍 слева снизу; Advisor справа снизу; меню ⋮ (смена модели на лету, rewind, редактор промптов).
3. Нормальный менеджер сейвов (список, rename/delete/duplicate, загрузка любого, не `saves[0]`, без `prompt()`).
4. SSE-прогресс для очереди действий и генерации мира (BalanceAgent шлёт прогресс по странам), `VITE_API_URL` вместо хардкода.
5. Настройки в UI: провайдеры/ключи/модели per-механика, язык контента, бюджет контекста.
6. Опционально: Game Master (чит-чат «сделай X»), Catalysts (сюжетные линейки), async-мультиплеер по ссылке — только если очень хочется, в самом конце.

## 7. Ключевые архитектурные решения (шпаргалка)

- **Промпт-пайплайн (как у оригинала):** состояние игры → сборка промпта с `${HELPERS}` → JSON-схема → модель → валидация/ремонт JSON → движок исполняет эффекты → БД → SSE на фронт. Сырой текст — только для советника/нарратива.
- **ИИ по именам, движок по id.** Никогда не показывать ИИ координаты/id; резолвить имена локально.
- **Бюджет контекста — фича №1 для локальных моделей.** Консолидация + обрезка описания карты (только изменившиеся регионы + соседи) + компактное описание (без городов) — иначе локальная 8B-модель захлебнётся.
- **Сложность и «душа» мира — это промпты, не код.** Тексты difficulty-блоков и структура 12 промптов — на wiki.paxhistoria.co (Basic Prompt Editing, Difficulty); у клона уже есть их адаптации в `backend-nest/src/prompts/`.
- **Детерминированный движок = исполнитель и валидатор, не оракул.** Исходы решет LLM; движок проверяет ресурсы/соседство и применяет эффекты.
- **Снапшот перед каждым ходом** (уже есть) = дешёвые rewind/Intervene/save-anywhere.

## 8. Риски

1. **Локальные модели слабее в instruction-following JSON** — нужен ремонт JSON, structured output где доступен (Ollama `format: json`), и тесты на 2–3 целевых моделях.
2. **Скорость:** прыжок на 1 год — большой ответ; на локальной модели это минуты → стриминг + показ событий по одному становятся обязательными.
3. **Рефакторинг без тестов** — этап 0 не пропускать.
4. **Scope creep** (каталог пресетов «как у них», мультиплеер) — держаться рамки из раздела 5.
